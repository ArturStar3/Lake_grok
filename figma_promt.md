## РОЛЬ И ГЛАВНАЯ ЗАДАЧА

Ты — senior product designer. Спроектируй **удобный, интуитивно понятный и отзывчивый** интерфейс для desktop-first веб-приложения **InfoLake** (информационно-аналитическая система / электронная разведывательная сводка, сокращённо **ОР**).

**Цель дизайна:** улучшить UX и визуальную согласованность, **не теряя** GIS-плотность данных, скорость работы оператора и все существующие сценарии. Это **рабочий аналитический инструмент**, а не маркетинговый лендинг.

**Язык интерфейса:** русский.

**Бренд в UI:** продукт — **InfoLake**; на главном экране в шапке используется короткий знак «**И**» + логотип; в auth — подзаголовок «Информационно-аналитическая система».

---

## КОНТЕКСТ ПРОДУКТА

InfoLake объединяет в одном рабочем месте:

- **Интерактивную карту** (объекты, события, зоны, обстановка, страны).
- **Табличные панели** с фильтрами и выбором записей.
- **Формуляры объектов** (структурированные разделы, markdown, вложения).
- **События** с разной геометрией на карте.
- **Зоны действия** (радиусы, полигоны, зоны из ТТХ техники, пересечения).
- **Оперативную обстановку** с версионированием (таймлайн ревизий).
- **Справочники** и **администрирование пользователей**.

Данные ограничиваются **правами группы безопасности** и **списком стран**, доступных группе.

## ПОЛЬЗОВАТЕЛИ И ПРАВА

Права задаются **группами безопасности**, не отдельными «ролями» в UI.

**Уровни по модулям:** `none` / `read` / `write`.

| Модуль | Что даёт пользователю |
|--------|------------------------|
| `targets` | Объекты на карте и в таблице |
| `events` | События |
| `operational_situations` | Оперативная обстановка |
| `formular` | Формуляры объектов |
| `country_dossier` | Досье страны |
| `persons` | Персоналии |
| `equipment` | Техника на объектах / каталог |

**Дополнительные флаги группы:** удаление (`can_delete`), управление справочниками (`can_manage_reference`), управление пользователями (`can_manage_users`), одобрение регистраций.

**Статусы учётной записи:** ожидание одобрения (`pending`), активен (`active`), заблокирован (`blocked`); возможна обязательная смена пароля.

**Как это отражать в дизайне:**

- Вкладка «Оперативная обстановка» видна только при read на `operational_situations`.
- Кнопки создания/редактирования/удаления — при write (+ delete где нужно).
- «Справочники» — при `can_manage_reference`.
- «Управление пользователями» в меню пользователя — при `can_manage_users`.
- Показывать состояния «нет доступа» / read-only явно, без «мёртвых» кнопок без пояснения.

## КАРТА ЭКРАНОВ И РЕГИОНОВ UI

### Auth (отдельные полноэкранные экраны)

| Экран | Назначение |
|-------|------------|
| Вход | Логин, пароль, показать/скрыть пароль |
| Регистрация | Логин, ФИО, пароль×2, комментарий, требования к паролю |
| Забыли пароль | Запрос сброса (офлайн-сценарий, без email в UI) |
| Ожидание одобрения | Сообщение после регистрации |
| Смена пароля | Обязательная или добровольная смена |

### Главное приложение (после входа)

```
┌─ Header: логотип «И», меню пользователя ─────────────────────┐
│  Сменить пароль | Управление пользователями* | Выйти         │
├─ Основная зона (Formular) ───────────────────────────────────┤
│ ┌─ Левая колонка «ОР» ──┐  ┌─ Правая: карта + панель ──────┐ │
│ │ Заголовок, действия    │  │ Карта (Leaflet / вектор)      │ │
│ │ Вкладки данных         │  │ Панель «Инструменты» (мера)   │ │
│ │ Фильтры + таблицы      │  │ Нижняя зона Features          │ │
│ └────────────────────────┘  └──────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

**Fullscreen карта:** карта на весь viewport; **правый sidebar** (~34% ширины, max ~520px) с теми же вкладками, фильтрами и таблицами; FAB для сворачивания sidebar; кнопка выхода из fullscreen.

## ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ (ЧТО ДОЛЖЕН ПОДДЕРЖИВАТЬ ДИЗАЙН)

### 1. Карта

- Базовая карта + **переключаемые оверлей-слои** (гидрография, транспорт, аэродромы, рельеф, подписи, POI; группы с «включить все»; учёт minZoom).
- **Маркеры объектов:** SVG по стране/типу; **кластеризация**; hover и pin групп; клик → просмотр формуляра.
- **Связь таблица ↔ карта:** checkbox в таблице подсвечивает объект на карте.
- **Fly-to:** кнопка «к карте» у объекта / события / обстановки.
- **Alt+клик:** создание объекта с координатами (страна по полигону, если определима).
- **Ctrl+клик** (в режиме измерения): постановка точек измерения.
- **Клик по полигону страны (GeoJSON):** модальное **досье страны**.
- **Сброс состояния карты:** «Сбросить все» в меню «Инструменты» (выделения, фильтры, режимы).
- **Легенда зон действия** при активных зонах.
- **Hover/pin панели зон** — список зон под курсором / закреплённая панель.
- **Координаты курсора** — компактная pill-подпись на карте.
- **Масштабная линейка** (в fullscreen).

### 2. Измерения

- Включение из меню **«Инструменты»** → «Режим измерения».
- Ctrl+клик — точки; линия и подписи расстояний на карте.
- Панель **Features:** таблица точек (№, lat/lng, дистанция), итоговая длина, удаление точки.
- Подсказка: «Зажмите Ctrl…».

### 3. Объекты (targets)

**Вкладка «Объекты»:**

- Дерево/таблица: группировка **страна → типы**; checkbox; fly-to; клик по названию → просмотр; edit/delete по правам.

**Фильтры (сворачиваемая панель):**

- Страны (multi + поиск), типы (дерево с потомками), название.

**Создание объекта (`AddTargetModal`):**

- Страна, тип (дерево по стране), родитель, название, метка на карте, выбор SVG-маркера (поиск + превью), lat/lng, зоны действия (тип + радиус км).
- После сохранения — опциональный переход в редактор формуляра.

**Редактирование (`EditTargetModal`, вкладки):**

1. **Объект** — поля объекта; **зоны затопления** (сценарий, уровень воды, сезонность, подпись); обычные зоны (радиус или **полигон**: рисование на карте или ввод координат).
2. **Техника** — образцы из каталога + количество.
3. **Формуляр** — markdown-разделы + вложения.
4. **Персоналии** — список, порядок вверх/вниз, редактор персоны.

**Рисование полигона зоны на карте:** баннер/подсказка поверх карты; toolbar завершения контура.

**Просмотр формуляра (`FormularModal`):**

- Навигация по разделам; вложения/галерея; **дерево подчинённости** (lazy-load, fly-to / открыть); блок техники; персоналии; «Редактировать».

**Mass-editor формуляра:** иерархия разделов, Markdown, upload/delete изображений.

### 4. Персоналии

- ФИО, должность, информационные разделы (markdown + вложения), **фото**, **связи** с другими персонами (типы связей из справочника).
- Модальное чтение карточки персоны (lightbox для фото).

### 5. Техника

- На объекте: autocomplete образца, количество, удаление; детальная карточка образца; переход в справочник.
- На карте: зоны из **ТТХ развёрнутой техники** + ручные зоны объекта.

### 6. События

**Вкладка «События»:** фильтры + таблица + оверлеи на карте.

**Рисование на карте (toolbar):**

- Типы: **Точка | Окружность | Территория (прямоугольник) | Произвольная форма (полигон)**.
- Отмена точки, завершить контур, редактор координат, **«Далее»** → модалка сохранения.

**Модалка события:**

- Название, дата/время начала–конца, объект (текст), тип, страна, маркер события, цвет, описание (markdown), параметры геометрии.

**Фильтры:** название, dateFrom/To, timeFrom/To, страны, типы; сброс.

**Таблица:** checkbox, fly-to, название, дата, время, страна, тип, edit/delete.

### 7. Зоны действия

**Вкладка «Зоны действия»:**

- Иерархические фильтры по **странам** и **типам действий** (листья: ручная зона / параметры техники).
- Quick-select по типам и странам.
- **«Учитывать рельеф»** (LOS для радарных зон) + индикатор расчёта.
- **«Показывать точки пересечения»** → таблица пересечений (checkbox → маркеры на карте).
- Диагностика отсутствующих зон каталога (для админов/аналитиков).

### 8. Оперативная обстановка

**Вкладка (по праву read):** фильтры + таблица + **таймлайн ревизий** выбранной серии.

**Создание:**

- Рисование **нескольких территорий (мультиполигон)** на карте: «Завершить контур», «Добавить территорию», счётчик «Территорий: N», координаты, **«Далее»** → модалка.

**Модалка обстановки:**

- Название, описание (markdown), дата/время обстановки, цвет контура, затронутые страны (multi-autocomplete), комментарий к изменению.
- **Несколько территорий:** отдельный редактор координат на каждую; добавить/удалить (минимум одна).
- Режимы сохранения: создать; **исправление** (без новой версии); **новое состояние**; fork «на основе»; при edit — выбор режима.

**Таймлайн:**

- Выбор ревизии на карте; edit/delete ревизии; при нескольких выбранных обстановках — группировка.

**Панель деталей на карте:**

- Метаданные, markdown, компактный таймлайн, «Редактировать» / «Новое состояние».

**На карте:** все кольца одной ревизии одним цветом; клик открывает детали/фокус.

### 9. Досье страны

- Клик по стране → модалка: разделы досье + вложения; карточка «Заполненность формуляров»; переход к объектам страны.
- При праве на справочники — редактирование (название, ISO, цвет, разделы markdown, вложения; создание страны из ISO GeoJSON).

### 10. Справочники (модальное окно)

Вкладки:

1. **Вооружение и техника** → подвкладки: Образцы | Категории | Параметры ТТХ | Единицы.
2. **Типы объектов**.
3. **Типы зон действия** (цвет, режим зоны).
4. **Характеры связей** (для персоналий).

### 11. Управление пользователями

- Оверлей из меню Header: запросы сброса пароля; ожидающие одобрения (выбор групп + одобрить); список активных (смена пароля: автоген / вручную + флаг must_change).

## СУЩЕСТВУЮЩИЕ ПРИНЦИПЫ ОТОБРАЖЕНИЯ (BASELINE — УЛУЧШАТЬ, НЕ ЛОМАЯ)

Используй как отправную точку; в редизайне **систематизируй** в токены и компоненты.

### Layout и композиция

- **Split desktop:** левая колонка данных ~**2fr**, правая (карта + Features) ~**1fr**, gap ~10–20px, общая ширина контента ~**80vw**.

- **Fullscreen:** карта fixed 100vw×100vh; sidebar **~34%, max 520px**, полупрозрачный белый + blur; FAB toggle sidebar.
- **Floating UI на карте:** detail panel обстановки (top-right, ~360px), toolbars (bottom/top center), zone hover list (left), legend (left-bottom), coords (left-bottom pill).

- **Ошибки:** красный текст + banner `#fef2f2` / border `#fecaca`.

### Map-specific UI

- **Situation draw toolbar:** bottom-center, white card, min-width ~280, primary `#008dd2`.
- **Event draw toolbar:** top-center, кнопки инструментов ~36px; confirm/cancel семантика.

### Плотность и spacing

- Gaps **6–12px**; section padding **12–16px**.
- Ячейки таблиц: **4–8px** vertical, **6–8px** horizontal.
- Scrollbars sidebar: **6px**, thumb полупрозрачный.

### Responsive (сейчас — точечный)

- Главный split **без** mobile-first breakpoints — **desktop GIS**.
- Модалки и filter grid: collapse на **~520–768px**.
- В редизайне: предложи **tablet** (sidebar/stack) без потери карты как главного якоря.

### Состояния взаимодействия

- Hover: tint `#f0f8ff`; primary darker; icon **scale(1.05–1.1)**.
- Active/selected: primary fill + white text; row `#f0f8ff` + border primary.
- Focus (формы): ring 3px rgba primary.
- Transitions **0.15–0.2s**.

## ТРЕБОВАНИЯ К РЕДИЗАЙНУ В FIGMA

1. **Удобство:** частые действия (fly-to, фильтр, выбор на карте) — на минимальной глубине кликов; явные режимы рисования.
2. **Интуитивность:** связь таблица ↔ карта ↔ модалка визуально очевидна; режим fullscreen не «теряет» контекст вкладки.
3. **Отзывчивость:** desktop 1440+ как primary; tablet — карта + выезжающая панель данных; не обязательно phone для GIS-рисования.
4. **Единая design system:** color/text/spacing/radius/elevation tokens; согласовать auth и main app (один primary).
5. **Сохранить все сценарии** из раздела «Функциональные возможности» — не выкидывать вкладки, toolbars, таймлайн, мультиполигон, 4 типа геометрии событий.
6. **Не делать:** consumer landing, dashboard-карточки вместо таблиц, декоративный hero, перегруз emoji/pills.
7. **Карта — главный визуальный якорь** правой колонки; UI не перекрывает критичные controls без dismiss.

## КРИТЕРИИ УСПЕХА

Дизайн считается успешным, если оператор за **1–2 минуты** находит: объект на карте, включает зону, создаёт событие с полигоном, рисует обстановку из двух территорий и видит ревизию на таймлайне — **без обучения**, с меньшим числом визуальных «шумов», чем в текущем UI, но с той же информационной плотностью.
